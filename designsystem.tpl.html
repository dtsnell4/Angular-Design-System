Directives

Block Container

The Block Container directive is the basic building block of an AppCentral page. This directive creates a block level <div> element inside a Bootstrap row and full width column element. It takes an optional “header” attribute which will add an <h3> element for the title of the block. An example of the block container is this page. Inspect the white rectangle this text is in to see the markup.

In most cases this should be the first html on your page:

<block-container  header=" ' My Header ' ">
  <!—Your content goes here -->
</block-container>

Will produce the following markup:

<div class="row ng-isolate-scope" header="'Decision Letter'">
  <div class="col-md-12">
    <div class="block-flat">
      <div class="row no-margin-top ng-scope" ng-if="style.header.length > 0">
        <div class="col-md-12">
          <div class="header">
            <h3 class="ng-binding">My Header</h3>
          </div>
        </div>
      </div>
      <div class="row no-margin-top">
        <div class="col-md-12" ng-transclude="">
          <!—Your content will show up here -->
        </div>
      </div>
    </div>
  </div>
</div>

Button Container

The Button Container is a simple container element which usually holds a Left Buttons or Right Buttons container, or both.

The Button Container directive takes an optional pagelocation attribute of “top”. This will remove the top margin to avoid extra space above the buttons when they are the first elements in the block container.

Left Buttons

The Left Buttons directive will make a button container that will automatically float and style buttons on the left side of the page.

Right Buttons

The Right Buttons directive will make a button container that will automatically float and style buttons on the right side of the page.

The following is an example of how to add a complete buttons container with right and left button blocks to a page:

<buttons-container pagelocation="'top'">
  <left-buttons>
    <!—Your left buttons here -->
  </left-buttons>
  <right-buttons>
    <!—Your right buttons here -->
  </right-buttons>
</buttons-container>

This will produce the following markup:

<div class="row ng-isolate-scope no-margin-top" ng-class="{top:'no-margin-top'}[style.pagelocation]" pagelocation="'top'">
  <div class="col-md-12">
    <ng-transclude>
      <div class="pull-left pull-left-not-xs sm100p form-inline leftbuttons">
        <ng-transclude>
          <!—Your left buttons here -->
        </ng-transclude>
      </div>
      <div class="pull-right pull-right-not-xs sm100p form-inline rightbuttons">
        <ng-transclude>
          <!—Your right buttons here -->
        </ng-transclude>
      </div>
    </ng-transclude>
  </div>
</div>

Modal Container

Use the Modal Container directive to create uniform, consistent modals across your application. Based on the Bootstrap Modal JavaScript component, the modal can be shown with or withoput a header.

Add the following code to your page to ceate the button that calls the Modal Container Directive without a header.

<buttons-container>
  <left-buttons>
    <button type="button" class="btn btn-primary" data-toggle="modal" data-target="#modal1">Open Modal</button>
  </left-buttons>
</buttons-container>

Add the following code to create the modal itself.

<modal-container targetid="'modal1'">
  <modal-content>      
    <div class="text-center">
      <div class="i-circle info"><i class="fa fa-thumbs-up"></i></div>
      <h4>Content Title</h4>
      <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.</p>
    </div>
  </modal-content>
  <modal-footer>
    <div class="col-md-6">
      <button class="btn btn-block btn-default" data-dismiss="modal" ng-click="yourCancelFunction()">Cancel</button>
    </div>
    <div class="col-md-6">
      <button class="btn btn-block btn-primary" data-dismiss="modal" ng-click="yourActionFunction()">OK</button>
    </div>
  </modal-footer>
</modal-container>

The targetid attribute of the modal container becomes the id attribute of the resulting modal. This unique ID must match the data-target attribute of the button that opens the modal.

Filter Container

The Filter Container is used on pages with large amounts of data that is able to be filtered via various form controls. It organizes, styles, and can show or hide the filters as needed.

Add the following code around your filter/form set:

<filters-container toastrexpires="May 20, 2019">
  <!—Your form/filter inputs here -->
</filters-container>

This will produce the following markup:

<div class="filter-block closed ng-isolate-scope opened" toastrexpires="May 20, 2019">
  <button id="show-filters" ng-click="vm.activate()" class="btn btn-default btn-padding btn-sm">
    <i class="fa fa-plus hidden-md hidden-lg hidden"></i>
    <i class="fa fa-minus hidden-md hidden-lg"></i> 
    <i class="fa fa-chevron-up hidden-sm hidden-xs hidden"></i>
    <i class="fa fa-chevron-down hidden-sm hidden-xs"></i> Filters
  </button>
  <div class="filter-content" style="display: block;">
    <div class="row no-margin hidden-sm hidden-xs">
      <div class="col-xs-12 filter-header">
        <a href="" id="pin-filters" class="rotateLeft90 hidden-sm hidden-xs" ng-click="vm.pinFilters()">
          <i class="fa fa-thumb-tack" title="Pin Filters"></i>
        </a>

        <a href="" id="close-filters" class="hidden-sm hidden-xs" ng-click="vm.activate()">

</div>

Panel Container (Panels/Tiles/Cards)

The Panel Container directive creates one of three different styles of content sub-containers, a standard Bootstrap-style panel, a card (3 per row) or a tile (one per row).

The required paneltype attribute takes either “panel”, "tile”, or “card”.

Panel Content

The Panel Content directive creates the content area (or body) of the panel and styles it appropriately based on the type of panel.

Add the sameheight class to the Panel Content directive to make all the cards in a group have equal height.

Panel Footer

The Panel Footer directive will create a footer area for the panel, if desired, which can contain action buttons, additional information, etc.

To create a full card-style panel, add the following to your page:

<panel-container paneltype="'card'">
  <panel-content class="sameheight">
    <!—Your panel content here -->
  </panel-content>
  <panel-footer>
    <!—Your panel footer content here -->
  </panel-footer>
</panel-container>

Which will render the following markup:

<div paneltype="'card'" class="ng-isolate-scope
  <div ng-if="style.paneltype === 'card'" class="col-md-4 ng-scope">
    <div class="card nohover">
      <div class="panel panel-default panel-square" ng-transclude="">
        <div class="panel-body sameheight">
          <div class="row no-margin-top">
            <div class="col-md-12" ng-transclude="">
              <div class="row no-margin-top ng-scope">
              	<!—Your panel content will show here -->
              </div>
            </div>
          </div>
        </div>
      </div>
    <div class="panel-footer">
      <div class="row no-margin-top">
          <div class="col-md-12" ng-transclude="">
            <!—Your panel footer content will show here -->
          </div>
        </div>
      </div>
    </div>
  </div>
</div>

Accordion Container

Use the Accordion Directive to create standard web accordions with a variety of styles and methods. Accordions use the panel component for structuring content.

Example - Basic Accordion

This is an example of a basic accordion. Accordions are constructed using four directives. The <accordion-group> container contains all the accordions that should be grouped together and interact with each other. By default when an accordion is opened, the currently open accordion is closed.

<accordion-container> is the main container using the panel panel-default classes. Inside that go the <accordion-content> and the <accordion-footer> tags.

Open the next accordion to see sample code to create an accordion group.

 The accordion footer.

Add the following markup to the page to make an accordion. Then add your content inside the header and footer.

<div class="panel-group accordion accordion-semi" id="basic">
  <accordion-container header="'The Basic Accordion'" targetid="'basictarget1'" groupid="'basic'" startopen="true" ng-repeat="">
    <accordion-content>
      <-- Your main content goes here --> 
      .
      .
      .
    </accordion-content>
    <accordion-footer>
      <-- Your footer content goes here -->
    </accordion-footer>
  </accordion-container>
  <-- Additional accordion containers go here -->
  <-- OR add the 'ng-repeat' directive to the accordion contanier element -->
  .
  .
  .
<div>
If you don't need a footer, then simply omit the <accordion-footer> tags

The parameters required for creating an accordion for each of the directive tags are as follows:

header
This is the text (or html with some modification) that will appear in the accordion header.
groupid
This is the same ID as in the Accordion Group above.
targetid
This is an id unique to the page for this specific accordion. This ID allows an accordion heading to know which content to open when it is clicked.
startopen
Add this optional attribute and set it to true if you want an accorion panel to start opened.
color
Add one of the context colors to the header background. (see example below)
ui.sortable
This is the optional Angular directive to make accordions draggable/sortable. (see example below)
preview
Add this optional attribute to add content that is visible under the header when the accordion is closed. (see example below)
active
Set active="true" to add the .panel-header-active class to the panel header.
NOTE: If you want more than one accordion to be open at once, simply omit the groupid parameter.

Buttons

Use the button classes on an <a>, <button>, or <input> element. There are six main styles for buttons.

<!-- Standard button -->
<button type="button" class="btn btn-default">Default</button>

<!-- Provides extra visual weight and identifies the primary action in a set of buttons -->
<button type="button" class="btn btn-primary">Primary</button>

<!-- Indicates a successful or positive action -->
<button type="button" class="btn btn-success">Success</button>

<!-- Contextual button for informational alert messages -->
<button type="button" class="btn btn-info">Info</button>

<!-- Indicates caution should be taken with this action -->
<button type="button" class="btn btn-warning">Warning</button>

<!-- Indicates a dangerous or potentially negative action -->
<button type="button" class="btn btn-danger">Danger</button>

<!-- Deemphasize a button by making it look like a link while maintaining button behavior -->
<button type="button" class="btn btn-link">Link</button>

Tables

AppCentral tables have a standardized appearance. To create a table, start by adding the <table> element with the classes .no-border and .mobile-table to assure it looks appropriate on mobile. Then add the <thead> element with the classes .no-border and .mobile-hide. Add the optional .nowrap class if the content of the headers should never wrap lines. Finally, add the <tbody> element with the classes .no-border-x and “.no-border-y. 

<table class="no-border mobile-table">
  <thead class="no-border mobile-hide nowrap">
    <tr>
      <th>
        ...
      </th>
    </tr>
  </thead>
  <tbody class="no-border-x no-border-y">
    <tr>
      <td>
       ...
      </td>
    </tr>
  </tbody>
</table>

Toastrs

Angular Toastr is a library to provide stylized, non-blocking notifications to users. Toasters are created from the Javascript as opposed to being add to the html templates. There are four default styles:

app.controller('foo', function($scope, toastr) {
  toastr.success('Hello world!', 'Toastr fun!');
});

app.controller('foo', function($scope, toastr) {
  toastr.info('We are opened today from 10 to 22', 'Information');
});

app.controller('foo', function($scope, toastr) {
  toastr.error('You just checked in your gulp.config file', 'Error');
});

app.controller('foo', function($scope, toastr) {
  toastr.warning('Your computer is about to explode!', 'Warning');
});

Grid System

AppCentral uses Bootstrap’s grid system to be fluid and responsive. For a correct layout, you must first add a container with the class row to create horizontal groups of content. Inside the row, add the columns of content as required. Each column is a container with a specifier class that determines how the rows are laid out in various screen sizes.

The basic column class is .col-xx-# where “xx” refers to the viewport size: “xs” for extra small (<786 pixels), “sm” for small (<992). “md” for medium (<1200), and “lg” for large (>1200).

Bootstrap’s grid system scales up to 12 columns per row. The “##” refers to the number of 1/12th sections of the row the column should span. For example, in medium size a column that would span 1/3rd of the row would have a class of col-md-4. This is because it takes four 1/12th sections to make up one third of the row (i.e. 4/12 = 1/3). The class making a cloumn take up 50% of a row would be col-md-6 (6/12 = 1/2). 100% is col-md-12 (12/12 = 1).

For mare details check out the Bootstrap Grid System page.

<!-- Stack the columns on mobile by making one full-width and the other half-width -->
<div class="row">
  <div class="col-xs-12 col-md-8">.col-xs-12 .col-md-8</div>
  <div class="col-xs-6 col-md-4">.col-xs-6 .col-md-4</div>
</div>

<!-- Columns start at 50% wide on mobile and bump up to 33.3% wide on desktop -->
<div class="row">
  <div class="col-xs-6 col-md-4">.col-xs-6 .col-md-4</div>
  <div class="col-xs-6 col-md-4">.col-xs-6 .col-md-4</div>
  <div class="col-xs-6 col-md-4">.col-xs-6 .col-md-4</div>
</div>

<!-- Columns are always 50% wide, on mobile and desktop -->
<div class="row">
  <div class="col-xs-6">.col-xs-6</div>
  <div class="col-xs-6">.col-xs-6</div>
</div>

<div class="row">
  <div class="col-xs-12 col-sm-6 col-md-8">.col-xs-12 .col-sm-6 .col-md-8</div>
  <div class="col-xs-6 col-md-4">.col-xs-6 .col-md-4</div>
</div>
<div class="row">
  <div class="col-xs-6 col-sm-4">.col-xs-6 .col-sm-4</div>
  <div class="col-xs-6 col-sm-4">.col-xs-6 .col-sm-4</div>
  <!-- Optional: clear the XS cols if their content doesn't match in height -->
  <div class="clearfix visible-xs-block"></div>
  <div class="col-xs-6 col-sm-4">.col-xs-6 .col-sm-4</div>
</div>

Hidden Content

Force an element to be shown or hidden with the use of show and hidden classes. They are only compatible with block level toggling. 

<div class="show">...</div>
<div class="hidden">...</div>

Additionally, the invisible class can be used to toggle only the visibility of an element, meaning its display is not modified and the element can still affect the flow of the document.


Responsive Hidden Content

To hide content at certain screen resolutions, add the classes hidden-xs, hidden-sm, hidden-md, hidden-lg. They will be hidden only within their standard breakpoints.

To make elements that are normally hidden visible within a particular resolution breakpoint, use the visible-xx-* classes. In this context, xx refers to resolution size, and * refers to the display property value. Display values available are block, inline and inline-block.

<p class="visible-xs-block"><code>visible-xs-block</code></p>
<p class="visible-sm-inline"><code>visible-sm-inlin</code>e</p>
<p class="visible-md-inline-block"><code>visible-md-inline-block</code></p>
<p class="visible-lg-block"><code>visible-lg-block</code></p>
